# Typography scale

Use the shared typography utilities to keep headings and body copy consistent. The tokens live in `styles/typography.css` and are
available globally as class names.

- `text-display-lg`, `text-display-md` – hero headlines and marquee moments
- `text-heading-xl`, `text-heading-lg`, `text-heading-md`, `text-heading-sm` – primary headings, section titles, and card titles
- `text-body`, `text-body-sm`, `text-body-lg` – descriptive copy, captions, and supporting text
- `text-label`, `text-helper`, `text-caption` – form labels, helper text, and metadata

```tsx
<h1 className="text-heading-lg">Card headline</h1>
<p className="text-body-sm text-muted">Guidance and supporting copy live here.</p>
```

# Button variants and sizes

Buttons inherit motion tokens and offline awareness. Use the `variant` and `size` props for consistent treatment.

```tsx preview
<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="subtle">Subtle</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>
<Button variant="danger">Destructive</Button>
<Button variant="link">Link</Button>
```

```tsx preview
<Button size="xs">XS</Button>
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>
<Button size="xl">XL</Button>
<Button size="icon" aria-label="Refresh">
  <svg viewBox="0 0 20 20" fill="none" className="h-4 w-4">
    <path
      d="M5 4a7 7 0 1 1-1.12 8.53"
      stroke="currentColor"
      strokeWidth="1.5"
      strokeLinecap="round"
      strokeLinejoin="round"
    />
    <path d="M4 5.5V2H1.5" stroke="currentColor" strokeWidth="1.5" strokeLinecap="round" strokeLinejoin="round" />
  </svg>
</Button>
```

```tsx preview
<Button loading>Submitting</Button>
<Button variant="outline" loading>
  Saving
</Button>
```

# Card composition

Cards expose variant, elevation, and interactive props. Wrap forms with `asChild` to forward semantics.

```tsx preview
<Card interactive>
  <CardHeader>
    <CardTitle>Usage snapshot</CardTitle>
    <CardDescription>Metrics update hourly and reflect aggregated agent activity.</CardDescription>
  </CardHeader>
  <CardContent>
    <p className="text-body">Place any layout primitives or data visualizations here.</p>
  </CardContent>
  <CardFooter>
    <Button size="sm" variant="secondary">
      View report
    </Button>
  </CardFooter>
</Card>
```

# Form fields

Use the shared `Field` component to keep labels, helper text, and validation messaging aligned with the new input API.

```tsx preview
<Field label="Email address" helperText="We will never share your email." required>
  <Input placeholder="ops@example.com" />
</Field>

<Field label="About" helperText="Share a quick summary.">
  <Textarea rows={3} />
</Field>

<Field label="Webhook secret" helperText="32 character token" error="Secret has already been registered">
  <Input status="error" value="abc123" onChange={() => undefined} />
</Field>
```

Inputs and textareas support `variant`, `size`, and `status` props:

- `variant`: `outline` (default), `subtle`, or `ghost`
- `size`: `sm`, `md`, `lg`
- `status`: `default`, `success`, `error`

Combine these with helper text or inline validation for consistent UX across admin workflows.
