fix(buy-sell): Correct category filtering in search_businesses_nearby function

PROBLEM:
When users select a category from Buy & Sell and share their location,
they receive "No results" messages even when matching businesses exist
in the database.

ROOT CAUSE:
The search_businesses_nearby() function was filtering by the wrong column:
- Used: WHERE b.category = p_category
- Should use: JOIN with buy_sell_categories and filter by key

The 'category' column contains raw scraped values (e.g., "beauty_salon"),
but the function receives the category KEY from buy_sell_categories table
(e.g., "Salon"). This mismatch caused zero results.
